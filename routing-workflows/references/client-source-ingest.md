# Client Source Ingest

Use when the user provides messy client material and wants it turned into a brief, site content, sitemap, design direction, or implementation plan.

## Intent

Extract reliable project facts from mixed sources without inventing missing details.

## Sources

- WhatsApp exports,
- PDFs/DOCX files,
- existing websites,
- screenshots,
- voice-note transcripts,
- brand materials,
- pasted notes,
- analytics/audit exports.

## Workflow

1. Inventory all source materials.
2. Extract facts, offers, audience, constraints, claims, contact details, tone, and must-include content.
3. Preserve source language when relevant, especially Polish/English bilingual content.
4. Separate:
   - confirmed facts,
   - plausible inferences,
   - missing information,
   - contradictions.
5. Convert findings into the target artifact:
   - brief,
   - sitemap,
   - copy inventory,
   - content model,
   - design requirements,
   - implementation checklist.
6. Ask only for missing information that materially changes the outcome.

## Output

Report:

- sources reviewed,
- confirmed facts,
- missing/uncertain facts,
- produced artifact path or summary,
- recommended next action.

## Avoid

- treating inferred client facts as confirmed,
- losing bilingual nuance,
- burying missing data instead of surfacing it.
